## 开发使用规范

- 新功能开发前，先在 `product/` 确认产品规则，在 `engineering/` 确认技术约束。
- 涉及模型回复、角色人设、话术策略时，同步更新 `ai/`。
- 涉及拉新、付费、裂变、推送时，同步更新 `growth/`，并在实现前做合规检查。
- 需求发生变化时，先更新文档，再实现代码，避免产品规则散落在组件里。
