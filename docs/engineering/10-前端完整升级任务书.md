# 戏命师前端完整升级任务书

本文档用于把当前第一版 taste-skill 方向扩展为完整前端升级。其他线程按任务领取实现；主线程负责验收、微调和收口。

## 当前设计方向

Design read：中文 AI 命理游戏/聊天产品的主界面，面向移动优先的娱乐转化用户。视觉语言是“玄学戏剧感 + 可操作的游戏中控台”，不是普通 SaaS，也不是纯 landing page。

设计参数：

- DESIGN_VARIANCE：7
- MOTION_INTENSITY：5
- VISUAL_DENSITY：6

已确认的第一版方向：

- 中间聊天区是主舞台。
- 顶部、侧栏、右栏降噪，不抢主对话。
- 紫黑金保留，但减少无意义玻璃、发光、金边堆叠。
- 主操作区优先使用统一 icon 语言，减少 emoji 原型感。

## 全局执行规则

- 每个线程开工前先保存自己的 checkpoint patch。
- 不要重写业务逻辑，除非任务明确要求。
- 不要改动其他线程负责的文件，必要时先在交付说明里提出冲突。
- 移动端优先，桌面端补齐；至少检查 375x812、768x1024、1280x720。
- 视觉改造必须跑 `npm run build`，并说明 `npm run lint` 结果。
- 如果新增依赖，必须先说明原因和替代方案；默认不新增设计系统。
- 禁止重新引入大面积 emoji 按钮、AI 紫蓝泛光、同质化玻璃卡片。
- 保持现有紫黑金品牌基调，但每个模块只能有一个主焦点。

## 任务 1：聊天主流程完整升级

负责人建议：一个线程独立完成。

目标：把聊天体验从“能对话”升级成“命盘推演主流程”，提升输入、等待、回答、卡片 CTA 的连续感。

范围：

- `src/components/ChatWindow.tsx`
- 必要时少量调整 `src/styles.css`

要做：

- 完善空状态、加载态、错误态、发送中状态。
- 优化消息气泡排版，长文本、多段文本、卡片混排都要稳。
- 让 quick ask 更像“问事入口”，当前选项可横滑，但不能遮挡输入栏。
- 卡片消息统一视觉层级：reading、seal、daily、sub、earn、box、market、share 都应像同一产品里的卡片。
- 输入栏增加 disabled/focus/empty 状态，不产生布局跳动。

验收：

- 发送问题时不会重复提交。
- 发送中按钮和 loading 有明确状态。
- 长文本不溢出，移动端不横向撑破。
- 所有 card.kind 分支仍可正常渲染。
- `npm run build` 通过。

## 任务 2：移动端导航和菜单升级

目标：把侧边栏从“功能列表”升级为移动端产品导航，减少拥挤感，突出核心转化路径。

范围：

- `src/components/Sidebar.tsx`
- `src/components/TopBar.tsx`
- `src/lib/mock-data.ts` 中的导航文案可小幅调整

要做：

- 导航分组：核心、资产、增长、设置。
- 首屏保留 6 到 8 个高频入口，其余入口降低优先级。
- 修复移动端菜单高度、底部气运卡、滚动边界。
- 统一导航 icon，不再使用 emoji 作为主要图标。
- 当前项、可操作项、未开放项视觉要可区分。

验收：

- 375px 宽度下菜单不横向溢出。
- 菜单打开/关闭不影响聊天滚动位置。
- 每个已接入 modal 的入口仍能打开对应弹层。
- 未接入入口不能误导为可完整使用，需有弱提示或 disabled 风格。

## 任务 3：右侧桌面面板升级

目标：让桌面端右侧从信息堆叠变成“命师资产 + 今日运营”的辅助面板。

范围：

- `src/components/RightPanel.tsx`
- 必要时少量调整 `src/styles.css`

要做：

- 大师卡：头像、等级、好感、额度、铭文槽重新排版。
- 今日任务：任务完成态、奖励、下一步 CTA 更清楚。
- 限时事件：保留稀缺感，但降低刺眼红色和强边框。
- 桌面端 1280x720 下不应压迫中间聊天区。

验收：

- 1280x720 下右栏内容不明显截断。
- 右栏按钮 hover/focus 状态完整。
- 右栏视觉权重低于聊天主舞台。

## 任务 4：弹层和抽屉统一升级

目标：让充值、订阅、盲盒、市场、出马赚钱、分享、问安、封印弹层像一个完整商业产品体系。

范围：

- `src/components/Modals.tsx`
- `src/components/Modal.tsx`
- 相关 UI primitives 如确有必要可小改

要做：

- 所有 modal/drawer 使用统一标题区、说明区、主 CTA、次 CTA。
- 订阅、充值、盲盒要有清晰“首版不收款/模拟”提示，避免误导。
- 市场、出马赚钱要突出运营预览，不要像真实金融交易。
- 分享卡生成态、空态、成功态、失败态补齐。
- 弹层在移动端高度、滚动、关闭按钮位置必须稳定。

验收：

- 每个 modal 都能打开、关闭、重复打开。
- 375x812 下无内容被底部遮挡。
- ESC/遮罩/关闭按钮行为不冲突。
- `npm run build` 通过。

## 任务 5：视觉 token 和 CSS 收口

目标：把当前零散的工具类沉淀成稳定视觉 token，降低后续改造成本。

范围：

- `src/styles.css`
- 与 class 命名强相关的组件少量同步

要做：

- 整理并命名：surface、panel、chat-field、input、button、border、glow。
- 删除不再使用或重复表达的 CSS utility。
- 统一 radius 规则：按钮、卡片、输入、弹层各自固定尺度。
- 统一边框透明度和阴影强度，避免每个组件自己调一套。
- 加入 `prefers-reduced-motion` 降级。

验收：

- CSS 没有明显重复 utility。
- 页面视觉没有退回“全是玻璃卡片”。
- 动画关闭偏好下没有持续强动效。
- `npm run build` 通过。

## 任务 6：内容与转化文案微调

目标：让文案更像“戏命师产品”，少一点占位和原型味。

范围：

- `src/lib/mock-data.ts`
- `src/components/ChatWindow.tsx`
- `src/components/Modals.tsx`

要做：

- quick ask 文案更有场景：事业、姻缘、财运、今日避坑、解封命盘、出马赚钱。
- 导航 note 减少重复玄学形容，改成明确收益或状态。
- CTA 文案区分：查看、生成、模拟、申请、加入候补。
- 明确“不真实收款/模拟阶段”的提示位置。

验收：

- 文案不使用过多感叹号和空泛玄学词。
- 关键按钮动作清晰。
- 商业相关功能没有误导真实交易。

## 任务 7：响应式和可访问性修复

目标：把体验从视觉稿推进到可用产品。

范围：

- 全组件按问题点小改

要做：

- 检查移动端横向溢出。
- 所有 icon button 有 `aria-label`。
- 按钮 disabled 态不可点击且视觉可辨认。
- 文本颜色对比度不要过低，尤其是小字。
- 键盘 focus ring 不丢失。

验收：

- 375x812、768x1024、1280x720 三档截图通过。
- 控制台无 React key、hydration、资源加载错误。
- 基础键盘 Tab 顺序可用。

## 任务 8：最终验收与收口

负责人：主线程。

目标：合并各线程成果后统一检查、微调、消除风格分裂。

验收清单：

- `npm run build` 通过。
- `npm run lint` 无新增 error；已有 shadcn fast-refresh warning 可记录但不作为阻塞。
- 所有 modal/drawer 可打开关闭。
- 聊天发送、quick ask、分享生成、问安、出马申请不回归。
- 移动端无裸样式、无横向溢出。
- 桌面端中间聊天仍是主焦点。
- 视觉 token 一致：颜色、圆角、边框、阴影、icon 风格统一。

交付物：

- 最终变更说明。
- 验收截图：移动端主界面、移动端菜单、桌面主界面、一个弹层。
- 回退说明：列出 checkpoint patch 或 git diff 边界。
