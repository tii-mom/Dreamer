# 紫微戏命师 · 技术栈选型与架构

## 为什么不做微信小程序

作为个人开发者，小程序对这个项目的限制是**致命的**：

| 限制项       | 具体影响                                                              |
| ------------ | --------------------------------------------------------------------- |
| **支付资质** | 个人开发者无法开通微信支付，必须企业营业执照+对公账户+ICP备案         |
| **类目审核** | 命理/占卜/运势属于敏感类目，99%概率被拒，且需额外资质证明             |
| **裂变封禁** | 微信严禁诱导分享（"拉3人助力"直接触发封号），分享卡片必须跳转回小程序 |
| **跳转限制** | 无法自由跳转外部支付页、无法打开外部H5完成TON/crypto支付              |
| **内容风控** | AI生成的命理解读若触发关键词，小程序直接下架                          |

**结论**：用**纯Web（PWA）+ 多平台Bot**是最优解，完全掌控流量、支付和裂变。

## 推荐技术栈

### 前端（用户看到的网页）

| 技术                  | 用途     | 为什么选它                                                                    |
| --------------------- | -------- | ----------------------------------------------------------------------------- |
| **React 18 + Vite 5** | 核心框架 | 构建速度极快（比CRA快10倍），AI工具（Cursor/Codex）生成React代码最准确        |
| **TypeScript**        | 类型安全 | 减少AI生成代码的Bug，让AI帮你做类型约束                                       |
| **Tailwind CSS**      | 样式系统 | AI写Tailwind类名比写CSS文件更靠谱，一行代码搞定响应式                         |
| **shadcn/ui**         | 组件库   | 不是传统组件库，而是可复制代码的UI系统，AI生成的高级感最强，支持暗黑/赛博风格 |
| **Framer Motion**     | 动画     | 命盘生成、盲盒开箱、签文翻转的仪式感动画                                      |
| **html-to-image**     | 分享卡片 | 把命盘HTML/React节点直接生成PNG，用于朋友圈/小红书分享                        |
| **React Query**       | 数据获取 | 管理用户命盘、订单、每日运势的缓存和状态                                      |

**移动端体验**：配置为 **PWA（渐进式Web应用）**，用户"添加到主屏幕"后体验接近原生App，有独立图标、全屏显示、离线缓存。

### 后端（API + Bot网关）

| 技术                          | 用途      | 为什么选它                                                   |
| ----------------------------- | --------- | ------------------------------------------------------------ |
| **Node.js + Express/Fastify** | API服务   | 全栈JavaScript，AI工具生成前后端代码一致性最好，你维护成本低 |
| **Prisma**                    | 数据库ORM | TypeScript原生，AI生成数据库模型和查询语句极其方便           |
| **PostgreSQL**                | 主数据库  | 用户命盘、订单、戏命师属性、技能卡交易——关系型结构最清晰     |
| **Redis**                     | 缓存/会话 | 存储每日运势缓存、助力状态、绑定码（5分钟过期）              |
| **Bull MQ**                   | 任务队列  | 异步处理命盘生成、AI解读、每日定时推送任务                   |

### Bot统一接入层（核心）

不要为每个平台写独立代码，做一个**Message Gateway**：

```
用户手机浏览器（React PWA）
        ↓ HTTPS
    Cloudflare + Nginx（腾讯云香港/新加坡节点）
        ↓
    Node.js API（统一后端）
        ↓
    Bot Gateway（适配器模式）
        ↓
微信Bot（Wechaty/ComWeChat）  飞书Bot（OpenAPI）  钉钉Bot（OpenAPI）
```

**绑定流程设计**：

1. 用户网页点击"领取戏命师" → 选择平台（微信/飞书/钉钉）
2. 网页显示对应**二维码/添加链接**（微信扫码加Bot好友，飞书扫码添加应用）
3. 用户添加Bot后，Bot自动发送**6位绑定码**
4. 用户回到网页输入绑定码 → 完成`UserID`与`BotPlatformID`的关联
5. 后续所有交互：Bot推送消息（文字+图片），网页查看详细HTML命盘报告

## 各平台Bot接入方案

| 平台     | 接入方式                                                     | 稳定性                   | 个人开发者可行性                             |
| -------- | ------------------------------------------------------------ | ------------------------ | -------------------------------------------- |
| **微信** | **Wechaty + ComWeChat**（Windows微信协议）或 **企业微信API** | 中（有封号风险，需养号） | ✅ 可行，但需备用号轮换                      |
| **微信** | 微信公众号/服务号（开发者模式）                              | 高                       | ✅ 最稳定，但主动推送有限制（服务号每月4条） |
| **飞书** | 飞书开放平台 Bot（Webhook）                                  | **极高**                 | ✅ **强烈推荐**，API最开放，个人开发者零门槛 |
| **钉钉** | 钉钉开放平台 Bot（Stream/Webhook）                           | 高                       | ✅ 可行，企业内部应用审核宽松                |

**建议**：

- **主阵地**：飞书Bot（API最开放，无封号风险，卡片消息美观）
- **必做**：微信Bot（Wechaty个人号，用户基数大，但准备2-3个号轮换防封）
- **补充**：钉钉Bot（企业用户转化）

**关键**：网页端只做**展示和支付**，对话交互主要在Bot里完成。用户扫网页二维码加Bot，之后Bot每天主动发消息，用户点击消息里的链接跳回网页查看详细报告。

## 部署架构（复用你的现有经验）

你已有腾讯云+Cloudflare经验，直接复用：

```
域名（tai.lat 或新域名）→ Cloudflare（CDN+SSL）→ 腾讯云香港/新加坡服务器
                                    ↓
                            Docker Compose 部署
                                    ↓
                        ┌───────────┴───────────┐
                    Nginx (反向代理)          Certbot (SSL)
                        ↓
            ┌───────────┼───────────┐
        React Vite      Node.js API      PostgreSQL+Redis
        (静态资源)      (容器化)         (容器化)
```

**为什么选香港/新加坡节点**：

- 微信Bot协议、飞书API、钉钉API从大陆访问都顺畅
- 无需ICP备案（命理内容备案难过）
- 你已有腾讯云海外部署经验
